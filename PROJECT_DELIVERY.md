### RBAC 权限模型

```
GUEST (访客)
  ├─ read:capsule

USER (普通用户)
  ├─ create:capsule
  ├─ read:capsule
  ├─ update:capsule (仅自己的)
  ├─ delete:capsule (仅自己的)
  └─ unlock:capsule

ADMIN (管理员)
  ├─ create:capsule
  ├─ read:capsule
  ├─ update:capsule
  ├─ delete:capsule
  ├─ unlock:capsule
  ├─ create:user
  ├─ read:user
  ├─ update:user
  ├─ delete:user
  ├─ moderate:content
  ├─ review:comments
  ├─ manage:permissions
  └─ view:analytics
```

## 🔒 安全机制

### 1. 认证安全

- JWT 签名验证：HS256 算法
- Token 过期检查：24 小时 Access Token，7 天 Refresh Token
- Bearer Token：标准 HTTP 授权头

### 2. 权限安全

- 多层权限检查：API -> 权限检查器 -> 业务逻辑
- 权限越级防护：只有所有者和管理员可编辑
- 权限异常处理：PermissionDeniedException

### 3. 数据安全

- 信息隐藏：根据权限显示内容
- 访问控制：基于用户角色的可见性
- 审计预留：支持审计日志记录
